Garden Bot - Wiring Instructions
==================================

All MOSFETs on the PCB are N-channel low-side switches. They switch the
GND side of their load, not the positive side.

PCB HEADERS REFERENCE
---------------------
Header 1 (sensor): pin 34 | pin 35 | 3.3V | GND (switched)
Header 2 (sensor): pin 17 (unused) | pin 18 (unused) | 3.3V | GND (switched)
Header 3 (sensor): pin 36 (unused) | pin 39 (unused) | 3.3V | GND (switched)
RGB header: pin 33 | pin 25 | pin 26 (unused) — MOSFET-switched LED outputs.
  Pins 33 and 25 are repurposed for the water level probes below, wired
  directly to the bare ESP32 module pads, bypassing the onboard MOSFETs.
  This leaves only pin 26 available if a single-color LED is still wanted;
  full RGB is no longer possible on this header.
  (Pin 16 MOSFET switches the GND rail of all three headers together.
   3.3V on all headers is always on.)

Programming header: TX | RX | 3.3V | GND (permanent)


1. PUMP
-------
Pin 32 drives an N-channel low-side MOSFET. The MOSFET sits between the
pump and GND — it switches the ground side of the load.
A flywheel diode is already present on the PCB for this output.

  Pump wire 1  →  Battery positive
  Pump wire 2  →  PCB pin 32 MOSFET load terminal (drain)
  (PCB routes MOSFET source → GND internally)


2. MOISTURE SENSOR (capacitive, analog output)
-----------------------------------------------
Uses header 1. Pin 16 switches the header GND, cutting the sensor circuit
during sleep.

  Sensor VCC   →  Header 1, 3.3V
  Sensor GND   →  Header 1, GND (switched)
  Sensor AO    →  Header 1, pin 34


3. WATER LEVEL SENSOR (two bare wire probes)
---------------------------------------------
Two wires submerged near the bottom of the water tank. Probes alternate
roles each measurement: one drives HIGH while the other reads analog via
its internal pulldown, then they swap. Alternating which probe drives
avoids electrolysis/corrosion from sustained one-directional DC bias.

This requires both probe pins to support digital output AND analogRead.
GPIO 33 and 25 satisfy this, but on this PCB they are normally wired to
onboard MOSFET gates (RGB LED header) rather than exposed as raw GPIO —
the header itself only exposes the MOSFET-switched load side. To use
them as direct bidirectional probe pins, wire to the bare ESP32 module
pads for GPIO33/GPIO25 instead of the RGB header.

  Probe wire A  →  ESP32 module pad, GPIO 33 (bypass RGB header)
  Probe wire B  →  ESP32 module pad, GPIO 25 (bypass RGB header)

  Note: GPIO 25 is an ADC2 pin, which can't be read while WiFi is active.
  This is fine here because readWaterLevel() runs in setup() before
  connectWiFi(). If the water level read is ever moved to run while
  WiFi is connected, this will break.

  Threshold: WATER_LEVEL_THRESHOLD in config.h is ~10% of the 12-bit ADC
  range (4095), tolerant of probe resistance up to several hundred kΩ —
  much more forgiving than the previous digital INPUT_PULLDOWN threshold,
  which needed resistance below the internal pulldown's value (~45kΩ,
  itself imprecise) to register as wet. The two directional readings are
  averaged (water_level_raw) and compared against the threshold; this
  trades away detection of one-sided probe corrosion (oxide layers can
  conduct asymmetrically) for a simpler single value, which is also
  published over MQTT each cycle to track drift over time.


4. TEMPERATURE SENSOR (DS18B20)
---------------------------------
VCC is driven directly by pin 13 (GPIO output, not a sensor header).
This allows the sensor to be fully powered down during sleep.
Pins 13 and 14 must be accessible on your PCB (test pads or solder points).

Required external component: 1x 4.7kΩ resistor (pull-up on data line)

  DS18B20 VCC   →  ESP32 pin 13
  DS18B20 GND   →  permanent GND (e.g. programming header GND)
  DS18B20 DATA  →  ESP32 pin 14
  4.7kΩ resistor between pin 14 (DATA) and pin 13 (VCC)
    (pull-up disappears during sleep when pin 13 goes LOW)


5. BATTERY VOLTAGE DIVIDER
----------------------------
Allows the ESP32 to measure battery charge level.
Uses pin 35 from header 1 as the ADC input.
ADC1 channel — no conflict with WiFi.

IMPORTANT: the GND pins on all sensor headers are switched by pin 16.
The 47kΩ resistor must connect to a permanent GND (e.g. programming
header GND or battery negative terminal directly), not a header GND.

Required external components: 1x 20kΩ resistor, 1x 47kΩ resistor

  Battery positive  →  20kΩ  →  junction
                                    |
                                 pin 35  (Header 1, pin 35)
                                    |
                                 47kΩ
                                    |
                              permanent GND  (battery negative)

  The 3.3V pin on header 1 is NOT used for this circuit.
  The voltage at junction (pin 35) will be at most ~2.95V at full
  charge (4.2V battery), safely within the ESP32 ADC input range.
